# 截图脱敏及展示指南

为了在保护公司隐私的同时最大化展示你的实力，请按照以下规范准备截图：

## 1. Commits 记录截图规范
- **路径：** `screenshots/commits/`
- **操作：**
  - 截取 Git 历史列表。
  - **必须遮盖：** 具体的项目路径、包含公司名的文件名、具体的业务逻辑描述。
  - **保留：** `feat:`, `fix:` 等前缀，提交时间，以及你作为提交者的头像。
- **目的：** 证明你的提交频率高、规范、且能有效管理 AI 生成的代码。

## 2. AI 对话截图规范
- **路径：** `screenshots/ai_chats/`
- **操作：**
  - 截取 Trae 或 OpenCode 的对话侧边栏。
  - **必须遮盖：** 代码中的敏感 API Key、公司数据库地址、具体的商业算法逻辑。
  - **保留：** 你对 AI 下达的复杂指令（Prompt）、AI 提供的架构建议、以及你对 AI 结果的 Review/修正过程。
- **目的：** 展示你如何“调教”AI，体现你的 Prompt Engineering 能力和架构思维。

## 3. 效能报告截图规范
- **路径：** `reports/`
- **操作：**
  - 使用 Trae 或 GitHub Copilot 的统计页面。
  - **保留：** 代码接受率、节省的时间百分比、提交次数统计。
- **目的：** 用数据量化你的 AI 原生开发水平。
